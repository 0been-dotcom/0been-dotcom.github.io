블로그를 시작하기로 마음먹었을 때
가장 먼저 떠오른 건 워드프레스였다.

검색하면 항상 상위에 나오고
유튜브 튜토리얼도 넘쳐났다.

근데 알아볼수록 걸리는 부분이 생겼다.

---

워드프레스로 블로그를 운영하려면
호스팅 비용이 발생한다.

저렴한 곳 기준으로도
월 3천원에서 1만원 사이는 나왔다.

1년이면 최소 3~4만원.

글을 쓰기도 전에 돈이 먼저 나가는 구조였다.

무료로 쓸 수 있는 wordpress.com 도 있긴 한데
애드센스 연결이 안 된다.

수익화를 목표로 한다면
유료 플랜으로 업그레이드해야 하는데
그러면 월 비용이 더 올라간다.


![screenshot](/assets/images/posts/2026-04-09-screenshot-1.png)


티스토리나 네이버 블로그도 봤다.

티스토리는 애드센스 연결이 되고
무료라는 점에서 괜찮았는데
구글 검색 노출이 워드프레스보다 불리하다는 얘기가 많았다.

네이버 블로그는 네이버 검색에서는 유리하지만
구글 애드센스 연결이 안 된다.

결국 구글 애드센스로 수익화를 하려면
선택지가 좁아졌다.

---

그러다 Jekyll 을 알게 됐다.

정적 사이트 생성기라는 설명을 처음 봤을 때는
개발자용 도구인 줄 알았다.

GitHub Pages 와 연동하면
호스팅 비용이 전혀 들지 않는다는 걸 알고
찾아보기 시작했다.

무료인데 구글 애드센스 연결이 된다.
github.io 주소도 구글 검색 노출에 문제없다.

이 두 가지만으로도 선택할 이유가 충분했다.


![screenshot](/assets/images/posts/2026-04-09-screenshot-2.png)


실제로 써보니 예상과 달랐던 부분이 있었다.

코딩을 몰라도 된다는 말이
처음엔 반신반의했는데

GitHub 에서 테마를 Fork 하고
저장소 이름만 아이디.github.io 로 설정하면
바로 블로그 주소가 생겼다.

터미널도, 서버 설정도 필요 없었다.
GitHub 웹사이트 안에서 전부 해결됐다.

---

워드프레스가 나쁜 도구라는 게 아니다.

기능이 많고 플러그인 생태계도 풍부하다.
디자인 자유도도 높다.

다만 비용 없이 시작하고 싶고
구글 애드센스로 수익화가 목표라면
Jekyll + GitHub Pages 조합이
현실적으로 가장 합리적인 선택이었다.


![screenshot](/assets/images/posts/2026-04-09-screenshot-3.png)


매달 나가는 호스팅 비용 없이
글 쓰는 것에만 집중할 수 있다는 점이
결정적이었다.

Jekyll 블로그를 만들고 나서
가장 먼저 든 생각은
과연 이 블로그가 구글에 검색될까 하는 것이었다.

아무리 글을 써도
검색이 안 되면 아무도 찾아오지 않는다.

구글에 노출시키는 방법을 찾아보면서
직접 해본 것들을 정리해봤다.

---

구글은 새로운 사이트를 자동으로 발견하기도 하지만
직접 등록하면 훨씬 빠르게 인식된다.

구글 서치 콘솔이라는 도구가 있다.

search.google.com/search-console 에 접속해서
구글 계정으로 로그인하면 된다.

---

서치 콘솔에 접속하면
속성 추가 화면이 나온다.

URL 접두어 방식을 선택하고
블로그 주소를 입력하면 된다.

https://아이디.github.io 형태로 입력한다.

그러면 소유권 확인 단계가 나온다.

---

소유권 확인 방법은 여러 가지가 있는데
HTML 파일 업로드 방식이 가장 간단하다.

구글에서 제공하는 html 파일을 다운로드해서
블로그 저장소 docs 폴더 안에 올리면 된다.

GitHub 저장소
-> docs 폴더
-> Add file -> Upload files

여기서 다운로드한 html 파일을 올리고
Commit changes 를 누른다.

빌드가 완료된 후
서치 콘솔에서 확인 버튼을 누르면
소유권 인증이 완료된다.

---

소유권 확인이 끝나면
사이트맵을 제출해야 한다.

사이트맵은 블로그 안에 있는 글 목록을 정리한 파일이다.

구글이 이 파일을 보고
어떤 글이 있는지 파악해서 색인을 생성한다.

Minimal Mistakes 테마는
jekyll-sitemap 플러그인이 기본으로 포함되어 있어서
별도로 만들 필요 없이

블로그주소/sitemap.xml 로 접속하면
자동으로 생성된 사이트맵이 보인다.

---

서치 콘솔 왼쪽 메뉴
-> Sitemaps 클릭
-> 사이트맵 URL 입력란에 sitemap.xml 입력
-> 제출 클릭

이렇게 하면 구글에 사이트맵이 등록된다.

---

사이트맵을 제출한다고 해서
바로 검색에 노출되는 건 아니다.

구글이 사이트를 크롤링하고
색인을 생성하는 데 시간이 걸린다.

빠르면 며칠, 늦으면 몇 주가 걸리기도 한다.

서치 콘솔에서
URL 검사 도구를 쓰면
특정 글이 구글에 색인됐는지 확인할 수 있다.

색인이 안 됐다면
색인 생성 요청 버튼을 눌러서
직접 요청할 수 있다.

---

구글 노출에 영향을 주는 요소가 몇 가지 있다.

글 제목과 본문에
실제로 사람들이 검색하는 키워드가 들어있어야 한다.

글 분량이 너무 짧으면
검색 결과 상위에 오르기 어렵다.

같은 주제의 글이 꾸준히 올라오는 블로그가
그렇지 않은 블로그보다 검색 노출이 잘 된다.

---

처음 블로그를 만들었을 때는
검색해도 내 블로그가 나오지 않았다.

서치 콘솔에 등록하고 사이트맵을 제출하고 나서
2주 정도 지나니까
일부 글들이 구글 검색 결과에 나타나기 시작했다.

당장 눈에 띄는 변화가 없어도
꾸준히 글을 올리는 게 결국 중요했다.

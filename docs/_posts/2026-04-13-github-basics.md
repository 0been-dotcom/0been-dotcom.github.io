Jekyll 블로그를 만들려면
GitHub 를 써야 한다는 걸 알고 나서
처음엔 막막했다.

개발자들이 쓰는 도구라는 인식이 있어서
어렵게 느껴졌는데
막상 해보니 기본적인 사용법은 단순했다.

---

GitHub 는 파일을 저장하고 관리하는 공간이다.

일반적인 클라우드 저장소와 다른 점은
파일의 변경 이력을 전부 기록한다는 것이다.

언제 어떤 파일을 바꿨는지
전부 남아있어서
잘못 수정했을 때 이전 상태로 돌아갈 수 있다.

블로그 용도로 쓸 때는
이 기능보다 GitHub Pages 가 더 중요하다.

저장소에 올린 파일을
그대로 웹사이트로 만들어주는 기능인데
별도 서버 비용 없이 무료로 쓸 수 있다.

---

GitHub 를 쓰려면 먼저 계정을 만들어야 한다.

github.com 에 접속해서
Sign up 을 클릭하면 된다.

이메일, 비밀번호, 사용자 이름을 입력하면
계정이 만들어진다.

사용자 이름은 나중에 블로그 주소가 되니까
영문으로 만드는 게 낫다.

아이디.github.io 형태가 블로그 주소가 되기 때문이다.

---

저장소는 파일을 담는 폴더 같은 개념이다.

계정을 만들고 나면
오른쪽 상단 + 버튼
-> New repository 를 클릭해서 만들 수 있다.

저장소 이름을 아이디.github.io 로 만들면
GitHub Pages 블로그용 저장소가 된다.

Public 으로 설정해야
GitHub Pages 가 무료로 작동한다.

---

저장소 안에서 파일을 만들거나 수정하는 방법은
GitHub 웹사이트에서 바로 할 수 있다.

파일을 클릭하면 내용이 보이고
오른쪽 상단 연필 아이콘을 누르면 편집 모드가 된다.

수정이 끝나면 아래쪽 Commit changes 버튼을 누르면
변경 사항이 저장된다.

Commit 은 변경 사항을 저장하는 행위다.

사진으로 치면 저장 버튼 같은 개념인데
변경 이력이 전부 남는다는 점이 다르다.

---

파일을 새로 만들 때는
저장소 메인 화면에서
Add file -> Create new file 을 클릭한다.

파일 이름을 입력하고
본문을 작성한 뒤
Commit changes 를 누르면 된다.

파일명에 슬래시를 입력하면
폴더가 자동으로 만들어진다.

예를 들어 docs/_posts/2026-04-14-글제목.md
이렇게 입력하면
docs 폴더 안에 _posts 폴더가 생기고
그 안에 파일이 만들어진다.

---

터미널에서 GitHub 를 쓰는 방법도 있다.

저장소를 로컬 컴퓨터로 가져오는 것을 clone 이라고 한다.

git clone 저장소주소

이 명령어를 실행하면
저장소 파일 전체가 로컬에 복사된다.

로컬에서 파일을 수정하고 나면
세 가지 명령어로 GitHub 에 올린다.

git add . 은 변경된 파일을 스테이지에 올리는 것이고
git commit -m "메시지" 는 변경 사항을 저장하는 것이고
git push 는 로컬 변경 사항을 GitHub 에 올리는 것이다.

처음엔 이 세 가지만 알아도 충분하다.

---

Fork 는 다른 사람의 저장소를 내 계정으로 복사하는 기능이다.

Minimal Mistakes 같은 테마를 적용할 때
원본 저장소를 Fork 해서
내 저장소로 가져오는 방식으로 쓴다.

Fork 버튼은 저장소 오른쪽 상단에 있다.

---

GitHub 를 처음 쓸 때는
용어가 낯설어서 어렵게 느껴지는데
실제로 하는 일은 단순하다.

파일을 만들고, 수정하고, 저장하는 것이다.

블로그 운영 목적이라면
웹사이트에서 직접 파일을 편집하는 것만으로도
충분히 쓸 수 있다.

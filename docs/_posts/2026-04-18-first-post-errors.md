Jekyll 블로그를 만들고 나서
첫 글을 올리기까지 생각보다 오래 걸렸다.

파일 하나 만들어서 올리면 되는 줄 알았는데
막상 해보니 오류가 연달아 나왔다.

겪은 순서대로 정리해봤다.

---

첫 번째는 _posts 폴더 위치 문제였다.

Jekyll은 글 파일을 _posts 폴더 안에 넣어야 인식한다.

Minimal Mistakes 테마를 Fork하면
저장소 루트에 _posts 폴더가 보이지 않는다.

루트에 직접 _posts 폴더를 만들어서 글을 넣었는데
블로그에 아무것도 뜨지 않았다.

한참 찾아보니
이 테마는 docs 폴더 안에 실제 블로그 파일이 있는 구조였다.

docs -> _posts 안에 파일을 넣어야
Jekyll이 글로 인식했다.

---

두 번째는 파일명 형식 문제였다.

Jekyll은 _posts 안의 파일 이름이
특정 형식이어야만 글로 처리한다.

형식은 연도-월-일-제목.md 다.

예를 들면 2026-04-20-첫번째글.md 이런 식이다.

날짜 형식이 틀리거나
확장자가 .md 가 아니면
Jekyll이 글로 인식하지 않는다.

파일명에 한글을 쓰면
URL이 인코딩되어 복잡하게 바뀐다.

나중에 관리가 불편해서
영문으로 바꾸는 게 낫다.

---

세 번째는 파일 상단 형식 문제였다.

Jekyll 글 파일은 맨 위에 front matter 가 있어야 한다.

--- 로 시작해서 --- 로 끝나는 블록 안에
layout, title, date, categories 같은 항목을 넣는 형식이다.

이 블록이 없거나 형식이 틀리면
Jekyll이 글로 처리하지 않는다.

title 값에 특수문자가 들어가면
빌드 자체가 실패하는 경우도 있었다.

따옴표로 감싸주니 해결됐다.

---

네 번째는 날짜 인식 문제였다.

오늘 날짜로 파일을 만들었는데
블로그에 글이 뜨지 않는 경우가 있었다.

GitHub Pages 서버와 한국 시간이 달라서
서버 기준으로 아직 미래 날짜로 인식하는 경우였다.

날짜를 하루 전날로 바꾸니 바로 표시됐다.

---

다섯 번째는 빌드 오류였다.

글을 올렸는데 사이트가 업데이트되지 않을 때
Actions 탭에서 빌드 로그를 확인하면 된다.

저장소 상단
-> Actions 탭 클릭
-> 가장 최근 항목 클릭
-> build 클릭

빨간 글씨로 오류 내용이 표시된다.

가장 많이 겪은 오류는 세 가지였다.

about.md 파일에서
이미 삭제된 샘플 글을 참조하는 오류,
Gemfile 버전 호환 문제,
front matter 형식 오류였다.

오류 메시지를 그대로 복사해서 검색하면
대부분 해결책이 나왔다.

---

이 문제들을 하나씩 해결하고 나서야
첫 글이 블로그에 정상적으로 떴다.

처음엔 오류가 날 때마다 크게 잘못된 것 같았는데
대부분 파일 위치나 형식 문제였다.

한 번 정상적으로 올라가는 흐름을 익히고 나면
이후 글 발행은 단순한 반복 작업이 됐다.

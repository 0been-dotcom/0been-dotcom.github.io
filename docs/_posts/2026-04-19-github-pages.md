GitHub Pages가 뭔지 처음 들었을 때
그냥 GitHub 안에 있는 뭔가 정도로만 이해했다.

실제로 써보니
내 저장소에 올린 파일을
그대로 웹사이트로 만들어주는 서비스였다.

별도로 서버를 구매하거나
호스팅 비용을 낼 필요가 없었다.

GitHub 계정만 있으면 됐다.

---

GitHub Pages를 쓰기 전에
저장소 이름 규칙을 먼저 알아야 한다.

GitHub Pages에는 두 가지 방식이 있다.

하나는 사용자 페이지 방식이고
다른 하나는 프로젝트 페이지 방식이다.

사용자 페이지 방식은
저장소 이름을 GitHub아이디.github.io 형태로 만드는 것이다.

예를 들어 GitHub 아이디가 example이면
저장소 이름을 example.github.io 로 만들면 된다.

이 경우 블로그 주소가
https://example.github.io 로 생성된다.

프로젝트 페이지 방식은
저장소 이름을 아무렇게나 지어도 되는데
주소가 https://example.github.io/저장소이름 형태가 된다.

블로그 용도라면 사용자 페이지 방식이 낫다.
주소가 깔끔하고
나중에 커스텀 도메인 연결도 수월하다.

---

저장소가 준비되면 GitHub Pages 설정을 시작한다.

GitHub 저장소 접속
→ 상단 탭 Settings 클릭
→ 왼쪽 메뉴 스크롤
→ Code and automation 섹션 아래 Pages 클릭

Pages 화면이 열리면
Build and deployment 항목 아래
Source 드롭다운이 있다.

Deploy from a branch 를 선택하면
아래에 브랜치와 폴더를 고를 수 있는 항목이 생긴다.

브랜치는 master 또는 main 중
본인 저장소에 있는 브랜치 이름으로 선택한다.

새로 만든 저장소는 대부분 main 이고
Fork한 저장소는 master 인 경우가 많다.

모르면 저장소 메인 화면에서
왼쪽 상단 브랜치 표시를 확인하면 된다.

폴더 선택에서는
/ (root) 와 /docs 중 하나를 고른다.

Minimal Mistakes 테마를 Fork해서 쓴다면
블로그 파일이 docs 폴더 안에 있어서
/docs 를 선택해야 한다.

/ (root) 를 선택하면
사이트 파일을 찾지 못해서
빈 페이지가 뜨거나 README 내용만 보이게 된다.

선택 완료 후 Save 클릭.

---

Save를 누르면
Pages 설정 화면 상단에
배포 주소가 표시된다.

https://아이디.github.io 형태다.

바로 열어보면 안 보이는 경우가 있는데
배포가 완료되기까지 시간이 걸리기 때문이다.

저장소 상단 탭
→ Actions 클릭
→ 가장 최근 워크플로우 항목 클릭

여기서 배포 진행 상황을 실시간으로 볼 수 있다.

초록색 체크 아이콘이 뜨면 배포 완료,
빨간색 X 아이콘이 뜨면 오류가 있다는 뜻이다.

오류가 생겼을 때는
해당 항목을 클릭하면 로그가 펼쳐진다.

빨간 글씨로 표시된 오류 메시지를
그대로 복사해서 검색하면
대부분 해결 방법이 나왔다.

---

배포가 완료됐는데 CSS가 깨지거나
이미지가 안 보이는 경우가 있었다.

이건 _config.yml 설정 문제였다.

GitHub 저장소
→ docs 폴더
→ _config.yml 파일 클릭
→ 연필 아이콘(Edit) 클릭

파일 안에서 url 항목을 찾는다.

url : 에 실제 블로그 주소를 입력한다.

사용자 페이지 방식이라면
https://아이디.github.io 를 그대로 넣으면 된다.

baseurl 은 사용자 페이지 방식이면
비워두거나 "" 로 설정한다.

이 두 항목이 맞지 않으면
사이트는 열리는데 스타일이 전부 깨져 보이거나
링크를 눌렀을 때 404 오류가 뜨는 경우가 생긴다.

수정 후 Commit changes 클릭.

---

커스텀 도메인을 연결하고 싶다면
GitHub 저장소
→ Settings
→ Pages
→ Custom domain 항목에 구매한 도메인 입력

도메인은 가비아, 네임칩, Cloudflare 같은 곳에서 구매할 수 있다.

구매 후 도메인 DNS 설정에서
CNAME 레코드를 아이디.github.io 로 연결하면
커스텀 도메인으로 접속이 가능해진다.

같은 화면에서
Enforce HTTPS 체크박스를 켜면
HTTPS 적용도 자동으로 된다.

처음엔 커스텀 도메인 없이 시작해도 충분하다.
github.io 주소도 구글 검색 노출에는 문제없다.

---

무료 호스팅이라고 해서
뭔가 제한이 많을 것 같았는데
실제로 쓰면서 불편한 적은 없었다.

월 100GB 트래픽까지 무료고
저장소 용량은 1GB 권장이지만
텍스트 위주 블로그라면 충분한 수준이다.

글을 쓰고 GitHub에 올리면
몇 분 안에 사이트에 반영된다.

서버 관리할 필요도 없고
비용도 없으니
개인 블로그 용도로는 나쁘지 않은 선택이었다.

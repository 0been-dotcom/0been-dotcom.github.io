Jekyll과 Minimal Mistakes 테마를 활용하여 간단하고 미니멀한 블로그를 구축해보았다. 시작하기 전에 Ruby부터 설치가 필요했는데, Terminal에서 'brew install ruby' 명령어로 쉽게 설치했다. 이후 'ruby -v'로 버전을 확인하여 제대로 설치되었는지 확인했다. 다음으로 Jekyll 설치를 위해 'gem install jekyll' 명령어를 사용했다. 'jekyll -v'로 Jekyll의 버전을 체크했더니 문제없이 설치되었음을 알 수 있었다.

GitHub에 새로운 리포지토리를 만들고 이곳을 블로그의 호스트로 설정했다. Minimal Mistakes 테마를 이용하려면 공식 Minimal Mistakes GitHub 리포지토리를 클론 해야 했는데, 리포지토리를 클론한 후에 자신의 리포지토리에서 Fetch 및 Pull 명령어로 변경 사항을 업데이트했다. 이렇게 하면 테마가 최신 상태로 유지되었다.

로컬 개발 환경에서 블로그를 미리보기 위해 'bundle exec jekyll serve'로 로컬 서버를 실행했다. 이를 통해 'localhost:4000'에서 블로그를 확인할 수 있었다. 스타일을 수정하려면 'assets/css/main.scss' 파일을 직접 조정해야 했고, 또 필요에 따라 여기서 스타일을 커스터마이즈했다. 

레이아웃을 바꾸고 싶을 때에는 '_layouts' 폴더 내의 파일들을 수정했다. 'default.html' 파일을 수정하거나, 'post.html'에서 각 포스트의 레이아웃을 조정할 수 있었다. 또 '_data/navigation.yml' 파일을 사용하여 원하는 메뉴를 구성했다. 블로그 내용에 따라 다른 페이지를 추가하고 싶을 때에는 '_pages' 디렉토리에 새로운 마크다운 파일을 생성하여 추가했다. 이 파일들에 front matter를 설정하여 각 페이지의 제목과 레이아웃까지 관리할 수 있었다.

페이징 기능을 추가하기 위해 'jekyll-paginate-v2' 플러그인을 설치하고 '_config.yml' 파일에서 페이지네이션 옵션을 설정했다. 이 옵션들을 통해 페이지당 표시할 포스트의 수를 조정할 수 있었다. 카테고리별로 포스트를 페이징 하려면 각 카테고리에 맞는 레이아웃을 생성하고 pagination 변수를 적절히 설정했다.

마지막으로 GitHub Actions를 통해 자동 빌드를 설정했다. '.github/workflows/deploy.yml' 파일을 생성하고 코드를 작성하여 블로그 업데이트 시 자동으로 배포되도록 했다. 이 과정에서 문제가 발생할 수도 있으니 log 파일을 항시 확인하여 오류를 수정했다. 

이처럼 Jekyll과 Minimal Mistakes 테마로 구성한 블로그는 설치와 구성 그리고 커스터마이즈까지 비교적 간단하게 진행할 수 있었다. 각 단계마다 변수들을 잘 관리하고 경로를 정확히 설정하면, 원하는 대로 깔끔한 블로그를 구축할 수 있었다.

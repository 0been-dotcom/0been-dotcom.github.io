Jekyll 블로그를 Github Pages에서 활용하기 위해서는 몇 가지 초기 설정이 필요했다. 특히, Minimal Mistakes 테마를 사용할 경우 기본적인 절차를 지키는 것이 중요하더라. 순서에 따라 진행하며 발생할 수 있는 문제를 사전 방지하는 방법이 있다.

1. Ruby 및 Jekyll 설치

   Ruby가 설치되어야 Jekyll을 사용할 수 있었다. Ruby 설치 후 터미널에서 ```gem install jekyll bundler```를 실행했다. 설치가 완료되면 ```ruby -v```와 ```jekyll -v```로 설치가 제대로 되었는지 확인할 수 있었다.

2. Minimal Mistakes 테마 다운로드

   Minimal Mistakes 테마는 Github 저장소에서 찾을 수 있었다. ```git clone https://github.com/mmistakes/minimal-mistakes``` 명령어로 로컬에 복제했다. 필요한 구성 파일을 로컬에 설정하는 것이 단계의 핵심이었다.

3. Jekyll 설정 파일 구성

   ```_config.yml``` 파일을 확인하고 필요한 부분을 수정했다. Minimal Mistakes 문서에서 제공하는 기본 설정을 토대로, 사이트 이름과 설명, 기본 URL 등을 수정했다. 배포 환경에 맞춰 몇 가지 추가 설정이 필요하다는 부분이 중요했다.

4. 커스텀 스타일 수정

   스타일 변경을 위해 ```assets → css → main.scss``` 경로에 있는 파일에서 필요한 변경을 했다. 이 파일을 수정하면 스타일 변경이 용이했다. 추가적으로, 자체적인 스타일 파일을 생성해 ```assets → css``` 경로에 추가할 수도 있었다.

5. 로컬 개발 환경 구성 및 테스트

   ```bundle exec jekyll serve``` 명령어를 통해 로컬 서버를 실행했다. ```http://localhost:4000```에서 블로그가 어떻게 보이는지 직접 확인할 수 있었다. 이 과정에서 발생하는 오류 메시지는 대개 미설치된 플러그인이 원인이더라. ```Gemfile```에 필요한 플러그인을 추가하고 ```bundle install```을 통해 문제를 해결할 수 있었다.

6. 콘텐츠 영역 크기 조정

   콘텐츠 영역의 크기를 수정하고 싶으면, CSS 파일의 ```max-width``` 값을 변경했다. 일반적으로 ```1200px``` 정도로 설정되어 있었고, 필요에 따라 이 값을 조정했다. 변경 후에는 로컬 서버에서 스크린 사이즈 중 하나에서 레이아웃이 깨지는 것을 미리 확인할 수 있었다.

7. 기타 플러그인 설치 및 설정 변경

   paginated v2 플러그인을 설치해 필요한 곳에 페이지네이션을 적용할 수 있었다. ```_config.yml``` 파일에서 플러그인에 필요한 설정을 간단히 변경하고, 사이트 구조를 바꾸면 보다 정돈된 레이아웃을 만들 수 있었다.

이와 같은 절차를 통해 Jekyll 블로그 Minimal Mistakes 테마를 성공적으로 설정할 수 있었다. 각 단계를 세심하게 따라가면 대부분의 오류를 방지할 수 있었다.

Jekyll 블로그를 운영하면서 모바일에서도 쉽게 설정하고 싶었다. 노트북 앞에 앉아 작업하는 것도 좋지만, 이동 중에 스마트폰이나 태블릿으로 편리하게 블로그를 관리할 수 있다면 얼마나 좋을까?

내가 찾은 해결책은 리모트 환경에 의존하지 않고도 모바일에서 Jekyll 블로그를 설정할 수 있는 방법이었다. 오늘은 그 방법들을 하나씩 공유해 보겠다.

우선, Jekyll은 GitHub Pages와 조합하면 정말 강력해진다. 하지만 클라우드 기반이라면 로컬에서의 작업과는 약간 다른 접근이 필요하다. 특히 모바일 장치에서 작업을 한다면 몇 가지 준비가 필요하다.

먼저, 모바일에서도 GitHub 레포지토리를 관리할 수 있는 방법을 찾아야 했다. 다행히도 몇 가지 앱들이 이 부분을 도와줄 수 있었다. 대표적인 앱으로 `Working Copy`라는 IOS 전용 Git 클라이언트와 `Termux`라는 안드로이드 앱이 있다. 이 두 앱을 통해 GitHub에서 레포를 복제하고 수정된 파일들을 푸시할 수 있었다. 물론, 이 앱들 모두 무료 버전도 기능이 아주 훌륭하다.

하지만 그 전에, 동일 네트워크 상의 데스크톱이 필요하다. 이유는 Jekyll은 로컬에서 서버를 실행해야 하고, 모바일 장치가 이 서버에 접속할 수 있도록 하기 위해서다. 동일한 와이파이 네트워크에서 데스크톱과 모바일 장치가 연결되어야 한다. 그렇지 않으면 모바일 장치에서 로컬 Jekyll 서버에 접근하기가 어렵다.

![미리보기 이미지](https://0been-dotcom.github.io/desktop:0)

모바일에서 Jekyll을 설정할 때 또 하나의 중요한 점은 테마의 커스터마이징이다. Jekyll 테마는 다양하게 존재하지만, 모바일 우선 디자인인지 확인하는 것이 중요하다. 마음에 드는 테마들을 먼저 선택한 다음, 필요에 따라 CSS 나 JavaScript 파일을 다듬어야 한다.

테마 파일의 수정은 `Working Copy`나 `Termux`에서 이루어질 수 있으며, 이런 모바일 앱은 터미널 명령을 손쉽게 입력할 수 있도록 해준다. 예를 들어, Jekyll 서버를 실행시키기 위한 명령어는 다음과 같다.

```
bundle exec jekyll serve
```

이 명령어를 입력하면 로컬 서버가 진행 중임을 알 수 있다. 모바일 장치의 브라우저에서 `http://<로컬 IP 주소>:4000`에 접속하면 된다.

Jekyll 환경은 기본적으로 오픈 소스인 점이 마음에 들었다. 그렇기에 안드로이드 전용 앱인 `JekyllEx`가 많은 이에게 추천됐다. 한 번 설치하면 모바일 장치에서도 쉽게 블로그를 관리할 수 있게 된다.

![JekyllEx 화면](https://0been-dotcom.github.io/mobile:0)

마지막으로, Jekyll 모바일 설정이 얼마나 강력한지를 보여줬다. 모바일 환경에서도 손쉽게 블로그를 수정하고, 포스팅할 수 있다면, 어디서든 블로그 운영이 즐거워진다는 사실이다.

이제는 나도 길을 걷다 좋은 아이디어가 떠오르면 모바일로 바로 실행에 옮길 수 있다. 한 손에 커피, 다른 손에 스마트폰만 있다면, 언제 어디서든 블로그를 관리할 수 있게 된 것이다.

![GitHub 모바일 설정 예시](https://0been-dotcom.github.io/desktop:50)

이렇게 긍정적인 변화가 가능하게 하는 Jekyll과 GitHub의 조합은 참 매력적이다. 모바일 환경에서의 설정은 생각보다 더 간단하고, 강력하다. 덕분에 내 블로그 운영이 훨씬 수월하게 되었다. 다음에는 더욱 다양한 Jekyll 테마와 기능들을 시도해 볼 계획이다. 모바일에서도 충분히 가능하다니, 정말 뿌듯하다.

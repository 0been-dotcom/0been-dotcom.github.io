Jekyll로 블로그를 만들 때 가장 인기 있는 테마 중 하나는 Minimal Mistakes다. 이 테마는 깔끔한 디자인과 다양한 기능을 제공해 사용자들에게 많은 사랑을 받고 있다. Jekyll과 Minimal Mistakes 테마를 설정하고 활용하는 방법을 정리해보겠다.

우선, Jekyll을 설치해야 한다. Ruby가 설치되어 있다면, gem 명령어로 바로 설치가 가능하다. 터미널에서 gem install jekyll을 입력하고 설치가 완료되면 ruby -v와 jekyll -v를 통해 설치 여부를 확인해 볼 수 있다. Jekyll은 32bit 기반이므로 설치할 때 64bit(x64)가 아닌 32bit(x86) 버전을 사용해야 한다. 만약 64bit로 설치할 경우, 문제가 발생할 수 있다.

Minimal Mistakes 테마를 사용하려면 해당 테마의 GitHub 저장소를 먼저 포크 또는 클론해야 한다. GitHub에서 mmistakes/minimal-mistakes를 찾아 자신의 저장소로 복제한 후, 로컬 저장소로 가져오더라. GitHub Pages와 잘 연동되기 때문에 쉽게 호스팅할 수 있다. 

테마를 자신의 블로그에 적용하려면 assets -> css -> main.scss 파일을 자신의 프로젝트에도 동일하게 복사해야 한다. 대부분의 스타일링은 여기서 변화를 줄 수 있다. 특히 컬러나 폰트 조정을 원하는 경우 이 파일에서 직접 수정하는 것이 좋다.

블로그의 글이나 페이지, 카테고리 구조를 설정하려면 _config.yml 파일을 수정해야 하는데, 이 파일은 Jekyll의 설정을 다루는 중심이 된다. 블로그 제목이나 설명을 비롯해 메타데이터 관련 설정을 할 수 있다. 많은 경우 이 파일을 수정하며 직접적인 웹사이트의 변화를 확인할 수 있다.

또한 다양한 플러그인을 활용하면 기능을 더욱 확장할 수 있다. 예를 들어, Jekyll Paginate V2 플러그인을 설치하면 카테고리별로 페이지네이션을 구현할 수 있다. plugin과 관련된 설정은 _config.yml 파일에 추가되어야 한다. 플러그인 설치 후에는 pagination 변수를 설정하고, 카테고리 레이아웃에 적용해야 한다.

블로그 글 작성이나 페이지 생성은 마크다운 파일을 만들어 _posts 디렉토리에 저장하는 방식으로 진행된다. 파일 명은 일반적으로 “YYYY-MM-DD-제목.md” 형태로 하고, 파일 상단에는 Front Matter를 사용해 필요한 메타데이터를 입력한다. 이는 게시글의 제목, 날짜, 레이아웃 등을 정의하는데 필수적이다.

로컬 개발 환경에서 결과물을 확인하고 싶다면 jekyll serve 명령어를 사용하면 된다. 이렇게 하면 로컬 서버가 실행되고, 웹 브라우저에서 http://localhost:4000으로 접속하면 블로그의 변화를 실시간으로 확인할 수 있다.

이와 같은 과정을 통해 기본적인 블로그를 구축할 수 있지만, 더 깊이 있는 사용자 정의도 가능하다. 예를 들어 Google Analytics를 연동하거나 Disqus 같은 댓글 시스템을 추가할 수도 있다. 이런 외부 서비스와의 통합은 대부분 설정 파일에서 API 키 등을 입력해 간단히 해결할 수 있다.

기본적인 설정이 완료되면 자신만의 스타일로 블로그를 꾸밀 수 있다. 테마가 제공하는 다양한 레이아웃과 모듈을 활용해 개성 있는 블로그를 꾸미는 것이 가능하다. 특히 Minimal Mistakes는 반응형 디자인을 지원하므로 다양한 디바이스에서도 어울리도록 설정할 수 있다.

이러한 과정을 통해 Jekyll과 Minimal Mistakes 테마를 활용한 블로그 설정을 완전히 이해하고 자신만의 개발 블로그를 운영할 수 있다.

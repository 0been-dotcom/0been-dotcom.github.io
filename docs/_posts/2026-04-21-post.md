처음에 Jekyll을 알아봤을 때, 개발자들만 사용하는 어려운 툴인 줄 알았으나 막상 사용해보니 전혀 그렇지 않았다. Jekyll은 GitHub Pages와 함께 사용되어 간편하게 무료 블로그를 운영할 수 있는 방법을 제공하고 있었다.

먼저 GitHub 계정을 만들었다. 그러고 나서는 깃허브에서 마음에 드는 Jekyll 테마를 찾아내어 Fork 기능을 사용해 내 계정으로 복사하였다. 이렇게 하면 내가 갖고 있는 테마를 기반으로 블로그를 시작할 수 있게 된다.

블로그를 위해서는 몇 가지 소프트웨어 설치가 필요했다. 먼저, Ruby를 설치하고 이후에 Jekyll과 Bundler라는 도구를 추가적으로 설치해야 했다. 모든 설치 과정이 처음에는 복잡하게 느껴질 수 있지만, 따라갈 만한 많은 가이드라인과 자료가 깃허브에 준비되어 있어서 충분히 스스로 할 수 있었다.

테마를 Fork한 후, 로컬 멀신에 복사한 블로그 파일을 내려받고 적절한 설정을 해줘야 했다. 설정 파일에서 제목이나 설명 등을 원하는 대로 수정해 주면 된다. 마크다운 문법을 사용해 손쉽게 글을 작성할 수 있다는 점은 큰 장점이었다. 익숙하지 않으면 처음 몇 번은 적응하는데 시간이 걸리겠지만, 한 번 익숙해지면 그 간편함에 감탄하게 된다.


![image](/assets/images/posts/2026-04-21-image_1.png)


깃허브 레포지토리를 만들고, 거기에 내 블로그 파일들을 올려놓으면, 내 블로그는 곧바로 GitHub Pages를 통해 게시된다. 무료로 호스팅 되고, 사용자 도메인을 설정해서 사용하면 정말로 그럴듯한 개인 블로그가 탄생하는 셈이다.

블로그 운영에서 자주 필요한 변경 및 테스트 작업도 GitHub Pages의 미리보기 기능을 통해 로컬에서 확인할 수 있다. 그러니 실시간 변경 사항을 확인하면서 작업을 진행하기에도 매우 편리하다.

Jekyll과 GitHub Pages를 사용해보니, 그 전에는 어렵고 복잡할 줄 알았던 것이 막상 해보면 오히려 다른 방법들에 비해 더 효율적이란 생각이 들었다. 모든 파일과 설정이 오픈되어 있으므로 커스터마이징도 자유롭게 할 수 있다.


![image](/assets/images/posts/2026-04-21-image_2.png)


개인 블로그를 꿈꾸는 이들에게 Jekyll과 GitHub Pages는 빼놓을 수 없는 선택지 중 하나라고 생각된다. 무료로 고성능의 블로그를 운영할 수 있다는 점에서 많은 이들이 관심을 가질 만하다. 웹 호스팅에 지쳐가고 있었다면, Jekyll로의 전환은 좋은 탈출구가 되리라.

제작한 블로그가 정상적으로 작동하기 시작하면, 드디어 콘텐츠 작성에 몰두할 수 있다. 매번 사용하던 CMS와 다르게, Jekyll에서는 글을 작성하고 업로드하는 것도 상당히 다르다. 하지만 다르다고 해서 더 불편하다기보다는 새로운 시각에서의 블로그 운영이라는 점에서 도리어 흥미롭게 느껴졌다.


![image](/assets/images/posts/2026-04-21-image_3.png)


테마 선택의 폭이 넓고, 사용자 커뮤니티도 활발하기 때문에 문제가 생겼을 때 도움을 받을 수 있는 방안도 다양하다. 초기 세팅만 잘 마치면, 그 이후부터는 내가 작성하는 콘텐츠에 더 집중할 수 있는 환경이 만들어진다. 매력적인 Jekyll과 GitHub Pages, 한 번 경험해보면 아마도 나처럼 감탄하게 될 것이다.
